## ⚙️ Customizing Your Experience

### Opening Settings

**Method 1: Sidebar**
```
Bottom of sidebar → "Settings" button
```

**Method 2: Mobile**
```
Menu (☰) → Sidebar opens → "Settings"
```

### Choosing Your Role

**Why it matters:**
- AI tailors responses to your needs
- Different roles = different communication styles

**Available Roles:**

1. **Student** 🎓
   ```
   Select if: Learning, studying, education
   AI will: Explain concepts, teach step-by-step
   Example response: "Great question! Let me explain..."
   ```

2. **Professional** 💼
   ```
   Select if: Work, business, career
   AI will: Provide strategic advice, actionable insights
   Example response: "From a professional perspective..."
   ```

3. **Developer** 💻
   ```
   Select if: Programming, coding, tech
   AI will: Give technical solutions, code examples
   Example response: "Here's how to implement this..."
   ```

4. **Researcher** 🔬
   ```
   Select if: Research, analysis, investigation
   AI will: Analytical approach, systematic thinking
   Example response: "Based on research methodologies..."
   ```

5. **Creative** 🎨
   ```
   Select if: Art, design, creative work
   AI will: Think outside the box, artistic perspective
   Example response: "Here's a creative approach..."
   ```

6. **General** 🌟
   ```
   Select if: Unsure, varied needs
   AI will: Balanced, versatile responses
   Default role for new users
   ```

**How to Select:**
1. Click on your preferred role card
2. Card highlights in blue
3. Click "Save Changes" button
4. Settings saved!

### Adding Personal Context

**What is Context?**
```
Additional information about:
- What you're working on
- Your goals
- Specific needs
- Current projects
```

**Examples:**

**For Student:**
```
"I'm studying computer science, currently learning
data structures and algorithms. Preparing for exams."
```

**For Professional:**
```
"I'm a marketing manager at a tech startup. Working
on product launch strategies and customer acquisition."
```

**For Developer:**
```
"Full-stack developer working with React, Node.js,
and PostgreSQL. Building a SaaS application."
```

**How to Add:**
1. Type in "Additional Context" text area
2. Be specific and detailed
3. Click "Save Changes"
4. AI will reference this in responses!

## 📂 Managing Multiple Conversations

### Creating New Conversations

**When to create new conversations:**
- Different topics
- New projects
- Separate contexts
- Clean start

**How to create:**
1. Click "New Chat" button (sidebar top)
2. New conversation created
3. Old conversation saved
4. Fresh chat starts

### Switching Between Conversations

**Desktop:**
```
Sidebar → Click any conversation
→ Messages load instantly
→ Continue where you left off
```

**Mobile:**
```
Menu (☰) → Sidebar opens
→ Tap conversation
→ Sidebar closes
→ Chat loads
```

### Conversation Auto-Naming

**How it works:**
- First message you send = conversation title
- Automatically generated
- Shows first 50 characters
- Updates instantly

**Example:**
```
You send: "How do I create a React component?"
Title becomes: "How do I create a React component?"
```

### Deleting Conversations

**Warning:** Cannot be undone!

**How to delete:**

**Desktop:**
1. Hover over conversation
2. Trash icon (🗑️) appears on right
3. Click trash icon
4. Confirm deletion
5. Conversation removed

**Mobile:**
1. Swipe/tap conversation (varies by device)
2. Look for delete option
3. Confirm deletion

### Clearing Current Chat

**Difference from Delete:**
- Delete: Removes conversation entirely
- Clear: Keeps conversation, removes messages

**How to clear:**
1. Click "Clear" button (top right of chat)
2. Confirm action
3. Messages removed
4. Conversation still in sidebar

## 💾 Data Persistence

### What Gets Saved

**Automatically Saved:**
- ✅ All messages
- ✅ All conversations
- ✅ User settings
- ✅ Role selection
- ✅ Personal context
- ✅ Login status

**When It Saves:**
- After every message sent
- When you change settings
- When you create/delete conversations
- On login/logout

### Where It's Stored

**Your Browser's LocalStorage:**
- Stays on your device
- Not sent to servers
- Private to you
- Persists across sessions

**What This Means:**
- ✅ Privacy: Your data stays local
- ✅ Speed: Instant loading
- ✅ Offline: Works without internet (UI only)
- ⚠️ Browser-specific: Different browsers = different data
- ⚠️ Clear browser data = Lost conversations

### Accessing from Different Devices

**Same Browser, Same Device:**
```
✅ All data available
✅ Seamless experience
```

**Different Browser, Same Device:**
```
❌ Data not shared
Need to login again
Separate conversations
```

**Different Device:**
```
❌ Data not synced
Create new account OR
Wait for cloud sync feature
```

## 🔐 Security & Privacy

### Your Data

**What We Store:**
- Email (for login)
- Password (locally only)
- Name
- Messages
- Settings

**What We DON'T Store:**
- Credit cards
- Personal documents
- Sensitive information (unless you type it)

### Security Tips

✅ **Good Practices:**
- Use strong passwords
- Don't share login credentials
- Logout on shared devices
- Clear browser data when needed

❌ **Avoid:**
- Saving passwords in browser on public computers
- Sharing sensitive info in messages (if connecting to cloud AI)

## 🆘 Common Issues & Solutions

### "I can't see my messages"

**Possible Causes:**
1. Wrong conversation selected
2. Messages not loaded
3. Scroll position

**Solutions:**
1. Check sidebar - correct conversation?
2. Scroll down to bottom
3. Refresh page
4. Check if logged in

### "Sidebar won't open on mobile"

**Solutions:**
1. Look for hamburger menu (☰)
2. Tap it clearly
3. If still stuck, refresh page
4. Check screen orientation

### "Settings not saving"

**Solutions:**
1. Make sure to click "Save Changes"
2. Check browser localStorage is enabled
3. Try different browser
4. Clear cache and try again

### "AI not responding"

**Current Setup:**
- AI responses are simulated
- Should respond in 1-2 seconds

**If stuck:**
1. Check typing indicator showing
2. Wait a few more seconds
3. Refresh page if needed
4. Try new message
